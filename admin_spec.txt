658: Кто может

667: Корректор, редактор

669: published → archived

679: Экстренная публикация:
680: Флаг is_urgent = true

685: 2. Система комментариев
686: Модель данных (иерархия)
687: Комментарий:
688: ├── id, news_id, user_id
689: ├── parent_id (NULL = корневой, != NULL = ответ)
690: ├── content (текст, markdown-разметка: жирный, курсив, ссылки)
691: ├── status: pending / approved / rejected / deleted_by_user / deleted_by_moderator

693: ├── likes_count, dislikes_count
694: ├── is_pinned (закреплён редактором)

1231: РАНЖИРОВАНИЕ (Scoring)
1232: Score = w₁·ContentScore + w₂·UserScore + w₃·PopularityScore + w₄·FreshnessScore + w₅·DiversityScore + w₆·EditorialScore +w₇·ContextScore + w₈·BusinessScore
1233: Веса адаптивные: персонализируются под пользователя (коллаборативный 
1234: фильтр на уровне весов: если пользователь часто кликает на «свежие» новости, w₄ растёт для него)
1235: ВЫХОД: Персонализированная лента
1236: (50 элементов → фильтрация → 20 → ранжирование → 10)
1237: 1. Компоненты Score
1238: 1.1 ContentScore (контентная близость)
1239: TF-IDF + Word2Vec по тексту новости
1240: Шаг 1: Векторизуем профиль пользователя
1241: - Берём все просмотренные новости за последние 90 дней
1242: - Складываем TF-IDF вектора, взвешенные временем (recency decay)
1243: - Получаем UserContentVector: 300-мерный вектор
1244: Шаг 2: Векторизуем кандидата
1245: - TF-IDF + Word2Vec заголовка, lead, тегов
1246: - Получаем CandidateVector
1247: Шаг 3: ContentScore = cosine_similarity(UserContentVector, CandidateVector)
1248: - Нормализуем: 0.0 → 1.0
1249: Рубрики и теги (категориальные):
1250: Если пользователь подписан на рубрику:
1251:   ContentScore += 0.15
1252: Если тег новости есть в подписках пользователя:
1253:   ContentScore += 0.05 за каждый совпадающий тег (макс 0.25)
1254: Если автор в подписках:
1255:   ContentScore += 0.10
1256: 1.2 UserScore (коллаборативная фильтрация)
1257: User-User Collaborative Filtering (для пользователей с историей >20 просмотров)
1258: Шаг 1: Найти «похожих» пользователей
1259: - Для активного пользователя U: берём его вектор взаимодействий
1260:   (item_id × action_type: view=1, like=3, share=5, comment=4, save=4)
1261: - Считаем cosine similarity со всеми пользователями
1262: - Берём топ-50 nearest neighbors
1263: Шаг 2: Рекомендовать то, что смотрели соседи, но U не видел
1264: - CandidateScore = weighted_average(neighbor_ratings, similarity)
1265: - Нормализуем: 0.0 → 1.0
1266: Для новых пользователей (<20 просмотров):
1267:   → Fallback на ContentScore + PopularityScore + ContextScore
1268: Item-Item Collaborative Filtering (альтернатива для масштабирования):
1269: - Предвычисляем матрицу похожести новостей (cosine similarity по взаимодействиям)
1270: - Если пользователь смотрел новость A → рекомендовать похожие B, C, D
1271: - Эффективнее для больших баз (O(n_items²) предвычисляется offline)
1272: 1.3 PopularityScore (популярность)
1273: PopularityScore = (
1274:   0.4 × log(просмотры + 1) / max_log_views +
1275:   0.3 × (комментарии / max_comments) +
1276:   0.2 × (поделились / max_shares) +
1277:   0.1 × (среднее время чтения / max_read_time)
1278: )
1279: Нормализация: делим на максимум за последние 7 дней
1280: Пенализация за «вирусный шлак»:
1281: - Если просмотры высокие, но время чтения <15 сек → PopularityScore *= 0.3
1282: - Если комментариев мало относительно просмотров → штраф
1283: 1.4 FreshnessScore (свежесть)
1284: Функция затухания (decay):
1285: t = часы с момента публикации
1286: Если t < 2:    FreshnessScore = 1.0
1287: Если t < 6:    FreshnessScore = 0.9
1288: Если t < 12:   FreshnessScore = 0.75
1289: Если t < 24:   FreshnessScore = 0.6
1290: Если t < 48:   FreshnessScore = 0.45
1291: Если t < 72:   FreshnessScore = 0.35
1292: Если t < 168:  FreshnessScore = 0.25  (7 дней)
1293: Если t < 720:  FreshnessScore = 0.15  (30 дней)
1294: Иначе:         FreshnessScore = 0.05  (архив)
1295: Особые случаи:
1296: - «Тема дня» (editorial pick): FreshnessScore = 1.0 на 24 часа
1297: - Экстренная новость: FreshnessScore = 1.0 на 6 часов
1298: - «Вечное» (справочный контент): FreshnessScore фиксирован = 0.3
1299: 1.5 DiversityScore (разнообразие)
1300: Проблема: Алгоритм может «застрять» в одной рубрике (filter bubble)
1301: DiversityScore для кандидата C в контексте уже выбранных элементов S:
1302: 1. Рубрика: если в S уже есть 2+ новости той же рубрики → DiversityScore *= 0.5
1303: 2. Автор: если в S уже есть новость этого автора → DiversityScore *= 0.7
1304: 3. Теги: Jaccard similarity тегов C с S < 0.3 → DiversityScore *= 1.2 (поощрение разнообразия)
1305: 4. Медиа-тип: если в S все текстовые, а C — видео → DiversityScore *= 1.15
1306: Финальный DiversityScore: 0.0 → 1.0
1307: Алгоритм MMR (Maximal Marginal Relevance) для финальной выборки:
1308: MMR = λ × Relevance(C) − (1−λ) × max[Similarity(C, Sᵢ)]
1309: λ = 0.7 (баланс релевантности и разнообразия)
1310: - Для «консервативных» пользователей: λ = 0.8
1311: - Для «открытых» пользователей: λ = 0.6

1346: │   ContextScore *= 1.2 для: Экономика, Политика, Наука
1347: ├── 14:00–18:00 (после обеда): культура, спорт, развлечения
1348: │   ContextScore *= 1.2 для: Культура, Спорт, Афиша
1349: ├── 18:00–23:00 (вечер): длинные чтения, расследования, видео
1350: │   ContextScore *= 1.3 для: длинные форматы (>2000 слов), Видео
1351: └── 23:00–06:00 (ночь): только экстренные + «вечные» материалы
1352:     ContextScore *= 0.1 для всего, кроме is_urgent
1353: День недели:
1354: ├── Пн–Пт: приоритет новостям, бизнесу
1355: ├── Сб: культура, афиша, природа
1356: └── Вс: спорт, длинные чтения, фото
1357: Погода (если шторм/снегопад):
1358: ├── ContextScore *= 1.5 для: Происшествия, Транспорт, ЖКХ
1359: └── ContextScore *= 0.8 для: Афиша (отмены мероприятий)
1360: Геолокация:
1361: ├── Если пользователь из Холмска → ContextScore *= 1.2 для новостей с тегом «Холмск»
1362: ├── Если пользователь из Охи → аналогично
1363: └── По умолчанию: Южно-Сахалинск
1364: 1.8 BusinessScore (бизнес-логика)
1365: Платные продвижения объявлений:
1366: ├── VIP-объявление → BusinessScore = 0.9 (гарантированно в ленте, но не топ-1)
1367: ├── Выделенное → BusinessScore = 0.7
1368: └── Поднятое → BusinessScore = 0.5
1369: Рекламные интеграции (нативные):
1370: ├── Помечены «Реклама» → BusinessScore = 0.6
1371: └── Ограничение: не более 1 рекламной на 10 органических
1372: Премиум-пользователи (Sakhcom+):
1373: ├── BusinessScore для рекламы = 0 (не показываем рекламу)
1374: └── BusinessScore для «эксклюзивных» материалов = 1.2
1375: 2. Финальная формула ранжирования
1376: Для каждого кандидата C:
1377: BaseScore = (
1378:   0.25 × ContentScore +
1379:   0.20 × UserScore +
1380:   0.15 × PopularityScore +
1381:   0.15 × FreshnessScore +
1382:   0.10 × DiversityScore +
1383:   0.10 × EditorialScore +
1384:   0.03 × ContextScore +
1385:   0.02 × BusinessScore
1386: )
1387: # Адаптивные веса (персонализация весов):
1388: Если пользователь «новостной наркоман» (>50 просмотров/день):
1389:   FreshnessScore вес *= 1.5, PopularityScore вес *= 0.7
1390: Если пользователь «глубокий читатель» (среднее время >5 мин):
1391:   ContentScore вес *= 1.3, PopularityScore вес *= 0.6
1392: Если пользователь «социальный» (много комментариев):
1393:   PopularityScore вес *= 1.2, UserScore вес *= 1.2
1394: # Нормализация весов (чтобы сумма = 1.0)
1395: FinalScore = BaseScore × DiversityPenalty
1396: Где DiversityPenalty:
1397:   Если рубрика C уже представлена в выборке N раз:
1398:     penalty = 0.8^N
1399:   Если автор C уже представлен:
1400:     penalty *= 0.9

1426: ├── Получаем ранжированный список 50
